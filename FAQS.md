## Interview Questions

### Q13: How to prepare for coding interviews?
**A:** Follow this approach:
1. Strengthen fundamentals (Weeks 1-2)
2. Solve 50 easy problems (Weeks 3-4)
3. Solve 50 medium problems (Weeks 5-6)
4. Solve 20 hard problems (Weeks 7-8)
5. Practice talking through solutions (Weeks 9-12)
6. Mock interviews (Weeks 13+)

### Q14: How to prepare for system design interviews?
**A:**
1. Understand fundamentals (scalability, caching, DB)
2. Learn from system design primer
3. Study real architecture (Twitter, Instagram)
4. Practice 3-5 problems in detail
5. Mock interviews with explanation
6. Focus on trade-offs and reasoning

### Q15: How to handle not knowing the answer?
**A:**

- Think out loud

- Ask for hints
- Try to solve a simpler version
- Show problem-solving skills

## Technical Questions

### Q18: What's the difference between HashMap and ConcurrentHashMap?
**A:**
- **HashMap**: Not thread-safe, faster
- **ConcurrentHashMap**: Thread-safe, slightly slower
- Use HashMap for single-threaded
- Use ConcurrentHashMap for multi-threaded

### Q19: When should I use SQL vs NoSQL?
**A:**
- **SQL**: Structured data, relationships, ACID, queries
- **NoSQL**: Unstructured data, flexibility, scalability
- Twitter: SQL (relationships) + Cache (performance)
- Instagram: SQL + NoSQL (different needs)

### Q20: What's the difference between @Component and @Bean?
**A:**
- **@Component**: Auto-detected, used on classes
- **@Bean**: Manual creation, used on methods
- @Component: Convention over configuration
- @Bean: More control, external libraries
